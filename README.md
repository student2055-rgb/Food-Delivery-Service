# Food-Delivery-Service
## Пітч
Це веб-додаток для користувачів, який дозволяє швидко замовляти їжу онлайн. Користувач може обрати страви, оформити замовлення та отримати доставку. Сервіс економить час і спрощує процес покупки їжі.
## Проблема
Люди витрачають багато часу на приготування їжі або замовлення через телефон. Відсутність зручного онлайн-сервісу ускладнює процес замовлення.
## MVP
Основний сценарій:
- Користувач обирає страву
- Додає її в кошик
- Оформлює замовлення
## Стейкхолдери
- Користувач
Хоче швидко і зручно замовити їжу.
- Ресторан
Хоче отримувати більше замовлень.
- Кур’єр
Хоче отримувати замовлення для доставки.
- Адміністратор
Хоче контролювати роботу системи.
## Мета
Створити простий сервіс для онлайн-замовлення їжі.
## Ознаки успіху
- Користувач може оформити замовлення
- Замовлення зберігається в системі
- Дані передаються без помилок
## Сценарій користувача
Передумова: користувач відкрив сайт
Кроки:
- Обирає страву
- Натискає “Додати в кошик”
- Переходить у кошик
- Натискає “Оформити замовлення”
Результат: замовлення створено
## Критерії приймання
- Користувач може додати товар у кошик
- Замовлення створюється після підтвердження
- Дані замовлення зберігаються
- Відсутні критичні помилки
## План перевірки
- Перевірити додавання в кошик
- Перевірити оформлення замовлення
- Перевірити збереження даних
- Перевірити роботу без помилок
## Ризики
R-01: помилки в оформленні замовлення
Рішення: тестування
R-02: повільна робота
Рішення: оптимізація
R-03: складний інтерфейс
Рішення: спростити дизайн
## Definition of Success
- Користувач може оформити замовлення
- Система працює стабільно
- Немає критичних помилок
- Інтерфейс зрозумілий
